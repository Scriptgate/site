Firefox Sculpting

March 13, 2014

From Firefox 47 onwards, 3D view is no longer available!
Firefox Sculpting is a term used to describe nesting elements to create a 3D model that can be viewed by using the 3D View option in the Firefox browser. This option creates a 3D model of the website by giving each element a height and stacking nested elements on top of eachother. With the use of JavaScript we can translate 3-dimensional points into blocks by placing an element at the corresponding x,y-position and nesting elements in it to reach a given height.
For sculpting we use two functions. The first function adds a 3-dimensional point to a hashmap with x and y coordinate as key, and an array of z coordinates as value. We save the points this way since elements that have the same x and y-coordinate are nested in the same parent element. The second function translates the 3-dimensional points into the actual HTML-elements.

First function

  • We define the key by joining together the x and y-coordinate seperated by a comma.
  • We verify wether the key already exists in our hashmap.
  • Let us first assume our key doesn't exist, we create a new array and add the z-coordinate. Then we add our key with the array as value to our hashmap.
  • If the key already exists, we can be sure the value will be an array, and we can test wether the z-coordinate is already contained in the array. If this is not the case, the z-coordinate is added to the array.
The second function is divided into four parts. Defining the parent element, preperation of the loop, iterating over the z-values and adding the line to the content. This is done for each key, thus each unique x,y-couple, in the hashmap.

Defining the parent element

  • First we create a parent div with document.createElement("div").
  • We check if for a given x,y-couple the corresponding array of z-coordinates contains a point at 0 height. If this is the case, the parent element will need to be visible, and is given a width and height of 15 pixels. We use 15 pixels since it corresponds with the height an element is given in the Firefox 3D View. If the parent needs to invisible, it's given a width and height of 0px. Even though an x,y-couple might not have point at 0 height, we still need to nest the higher points in invisible elements.
  • We give the parent element an absolute position. That way we won't have to worry about the order in which the elements are created.
  • The parent element receives a left position equal to the x-coordinate of the point. The x-coordinate is extracted from the key with a split function with a comma as argument. This returns an array of the string split up each time a comma occurs. We know the key exists out of the x and y-coordinate seperated by a comma, so the x-coordinate is the first element of the returned array. Since the key is a string object we parse it to an integer and multiply it with 15, to correspond with the width of the elements. We do the same for the top position and use the second element from the returned array.

Preperation of the loop

  • We create a new variable parent that will hold the last nested element during the iteration.

Iteration over the z-values

  • Since we already created a parent element, visible or invisible for a given x,y-couple, we start the iteration at position 1 and continue to the highest number in the array of heights; which is given by the function Math.max.apply(Math, array).
  • We create a new element that will be nested in our parent variable.
  • We give this new element a width and height in the same way as the parent element, if the iterated value is in the height array, it is visible, if not, invisible.
  • We nest the element in the parent variable with the function .appendChild(child).
  • We set our parent variable to the newly created element, this way, when the iteration continues it will add new elements to the last nested element; creating a lineair nested element.

Adding the line

  • As a last step, we add the parent element, defined in variable a to an element in the body of our page.
